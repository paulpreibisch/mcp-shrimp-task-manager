# 任務恢復失敗

❌ **恢復狀態:** 失敗

**任務ID:** `{taskId}`

**錯誤訊息:** {message}

## 可能原因

1. **任務已存在**: 該任務可能已經在當前任務列表中
2. **備份不存在**: 在備份文件中找不到指定的任務
3. **ID 格式錯誤**: 提供的任務ID格式不正確
4. **權限問題**: 無法讀取備份文件或寫入任務文件

## 建議操作

- 使用 `get_deleted_tasks` 查看所有可恢復的已刪除任務
- 檢查任務ID是否正確 (應該是UUID格式)
- 使用 `list_tasks` 確認任務是否已經存在於當前列表中
- 如果問題持續存在，請檢查系統權限和備份文件完整性
